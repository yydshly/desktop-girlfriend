# MiniMax Execution

MiniMax 作为代码执行 Agent 的工作边界和约束。

---

## 执行边界

### 核心原则

| 原则 | 说明 |
|------|------|
| **单任务执行** | 每次只执行一个任务，不并发多个需求 |
| **不扩大需求** | 不自行扩大需求范围，保持最小改动 |
| **不跨阶段** | 不实现未来版本的能力 |
| **修改追踪** | 每次修改必须记录文件、测试和风险 |

### 允许的行为

- 根据当前版本（V0-V7）实现对应功能
- 修改已存在的模块内的代码
- 添加测试和文档
- 重构已有代码（保持行为不变）

### 不允许的行为

- 在未授权情况下接入外部 API
- 提交密钥、API Key、Token
- 提交真实人物素材（照片、视频、语音）
- 提交本地隐私文件
- 实现未来版本的能力

---

## 修改后必须说明

每次代码修改后，必须提供：

```markdown
### 修改文件
<!-- 列出所有变更文件 -->

### 测试命令
<!-- 执行的测试命令 -->

### 测试结果
<!-- 测试输出 -->

### 风险点
<!-- 可能的风险 -->
```

---

## 权限边界

| 操作 | 是否允许 |
|------|----------|
| 创建新文件 | ✅ 允许 |
| 修改现有文件 | ✅ 允许 |
| 删除文件 | ✅ 允许（需说明） |
| 接入 MiniMax API | ❌ 需授权 |
| 接入其他外部 API | ❌ 需授权 |
| 提交密钥 | ❌ 绝对禁止 |
| 提交隐私素材 | ❌ 绝对禁止 |
